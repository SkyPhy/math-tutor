Development Plan Summary: Claude Opus 4.8 Integration for Math Tutor
I've created a 7-week implementation plan for integrating Claude Opus 4.8 with your existing math tutoring system. Here's the high-level roadmap:

The Approach
Claude augments, SymPy verifies — Claude handles reasoning & explanation, SymPy remains the deterministic verification anchor
Separate chat streams — Claude chatbox operates independently alongside Socratic hints
Safety-first design — Every Claude response validated against your existing Policy Engine (SEPGA)
Graceful fallback — If Claude API fails, system seamlessly falls back to pure Socratic
7-Phase Implementation
Phase	Timeline	Focus
1	Weeks 1-2	Backend infrastructure (Claude service class, SDK, config)
2	Weeks 2-3	New FastAPI endpoints (/claude/chat, /claude/explain, etc)
3	Weeks 3-4	Prompt engineering with Socratic constraints + policy validation
4	Weeks 4-5	Frontend ClaudeChatbox component + tab UI for separation
5	Weeks 5-6	Integration, error handling, rate limiting, timeout strategy
6	Weeks 6-7	Unit/integration tests, pedagogical validation
7	Week 7+	Deployment, monitoring, cost tracking
Key Technical Decisions
When to use Claude vs Socratic:

Simple problems → Pure Socratic
Conceptual questions → Claude depth
High struggle (hint level 3+) → Offer Claude as enhancement
Session Model:

Each session has two separate conversation threads
Shared adaptive context from experience memory
Claude usage tracked (calls, tokens, cost)
Safety Guardrails:

System prompt includes: "NEVER give final answer directly; ALWAYS ask questions back"
ClaudePolicyValidator checks every response
If Claude violates Socratic constraint → regenerate or fallback
Error Handling:

10s timeout per Claude call
1 retry with exponential backoff
Circuit breaker: disable after 3 consecutive failures
Critical Files to Create
backend/claude_service.py — ClaudeService class, API integration, conversation history
backend/prompts.py — 4 system prompts (Socratic Mathematician, Concept Clarifier, Debugger, Misconception Corrector)
frontend/src/components/ClaudeChatbox.tsx — Chat UI, message rendering, input handling
backend/config.py — Environment config, secrets, API key management
Files to Modify
backend/main.py → Add /claude/* endpoints, extend ExperienceMemory for Claude conversations
frontend/src/App.tsx → Claude state, tab switching logic
backend/requirements.txt → Add anthropic SDK
Success Metrics
✅ Claude responses <10s latency

✅ 100% compliance with Socratic policy

✅ <$0.05 cost per student session

✅ Students engage with Claude chat 30%+ of sessions

✅ Zero pedagogical errors in Claude explanations

The plan is designed to integrate cleanly with your existing architecture—Claude doesn't replace anything, it augments the Socratic engine while SymPy remains the verification source of truth.
