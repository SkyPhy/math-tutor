## 0. 这一步要解决什么

用户的**核心诉求**（最高权重）：

> 从**解决学生的逻辑问题、优化其逻辑思维**出发——区别于传统"按知识点/题型刷题"的平台。
> 我们按**「难度」+「逻辑思维（解决问题）类型」**标签来分类与训练。

现状的矛盾：`exam.py` 现有的 **25 个标签全是"知识点"**（数的运算、平面图形、统计图……），正是愿景要
**超越**的那套分类。本设计**不删**知识点维度（它对"覆盖课标"仍有用），而是新增一个**正交**的
**逻辑思维类型**维度 + 一个**难度**刻度，让同一道题同时带：

```
知识点标签（已有，回答"考什么内容"）   ⟂   逻辑思维类型标签（新增，回答"练什么思维"）   +   难度（新增，0–9）
```

正交的意义：同一种思维（如"逆向倒推"）可出现在 `数的运算` / `货币` / `时间` 任意知识点上；
诊断学生时，我们关心的是**哪一类思维薄弱**，而不是哪一个知识点没背熟。

## 1. 难度刻度（目标 #2：可上可下，幼儿园 ↔ 大学）

一个 **0–9 的"推理深度"刻度**，**独立于年级/知识点**——锚名只是参照，同一逻辑类型可在任意档位出题。

| 档 | 锚点 | 特征 |
|----|------|------|
| 0 | 启蒙（幼儿园）| 直观、单步、具体实物 |
| 1 | 低年级 I | 单步、20 以内 |
| 2 | 低年级 II | 单步百以内 / 简单两步 |
| 3 | 中年级 I | 两步、引入解题策略 |
| 4 | 中年级 II | 多步、需选择策略 |
| 5 | 高年级 I | 分数/比例、抽象关系 |
| 6 | 高年级 II | 多步多策略组合 |
| 7 | 初中过渡 | 代数化、形式化推理 |
| 8 | 高中 / 竞赛初步 | 多策略嵌套 |
| 9 | 大学通识 / 奥赛 | 抽象建模、严密论证 |

> 与现有 `questions.grade`（1–6 年级）**不同**：`grade` 是"课标年级归属"，`difficulty` 是"思维难度"。
> 二者并存——一道 3 年级知识点的题，其逆向倒推版本难度可达 5。

## 2. 逻辑思维类型分类（草案 · 13 类 / 5 族）

每类给出：**思维动作**（这类题在练什么）+ **诊断信号**（学生在这类上的典型逻辑缺陷——
正是目标 5/6"指出学生逻辑缺陷"的判据）。均**与知识点无关**。

### 族 A · 推理模式
| 标签 | 思维动作 | 诊断信号（学生逻辑缺陷）|
|------|---------|----------------------|
| 归纳推理 | 从特例发现规律并推广到一般 | 只盯个例不会概括；据少数例错误归纳 |
| 演绎推理 | 由已知条件按规则逐步推出结论 | 跳步、漏用条件、因果倒置 |
| 类比迁移 | 把新问题映射到已掌握的模型 | 迁移错位；想不起可联系的旧知 |

### 族 B · 策略与转化
| 标签 | 思维动作 | 诊断信号 |
|------|---------|---------|
| 逆向倒推 | 从结果反向还原每一步 | 只会顺推、不会用逆运算还原 |
| 假设调整 | 先假设一种情形再按差异修正（鸡兔同笼）| 不会设假设；不会按差量调整 |
| 化归转化 | 化繁为简、化未知为已知 | 被表象困住，找不到等价的简单形式 |
| 整体思想 | 把一组量作为整体处理而非逐个 | 纠缠局部、看不到整体关系 |

### 族 C · 结构化枚举
| 标签 | 思维动作 | 诊断信号 |
|------|---------|---------|
| 分类讨论 | 按互斥且穷尽的标准分情况 | 漏情况、重复；分类标准混乱 |
| 有序枚举 | 有序、不重不漏地列出所有可能 | 无序乱列、漏数或重数 |

### 族 D · 表征建图
| 标签 | 思维动作 | 诊断信号 |
|------|---------|---------|
| 数形结合 | 用线段图/示意图表征数量关系 | 不会画图、读不懂图；图意与题意不符 |
| 列表对应 | 用表格/一一对应整理信息 | 信息散乱、对应错位 |

### 族 E · 建模与估计
| 标签 | 思维动作 | 诊断信号 |
|------|---------|---------|
| 方程建模 | 设未知数、列等量关系求解 | 不会设元；等量关系列错 |
| 估算逼近 | 用上下界/近似判断结果合理性 | 缺数感；不检验结果是否合理 |

> **这是草案，分类本身是产品决策**——增删/改名/并族请直接改本表与 `exam.py::LOGIC_CATALOGUE`
> （二者是同一份事实的文档面与代码面，改完跑 §5 的导入自检即可）。

## 3. 怎样接进现有系统（数据模型，已就绪）

存储层**无需改 schema** 即可承载：`question_tags(dimension, subdimension, tag)` 已支持任意维度。

- 知识点标签：`dimension=核心知识领域 / 综合与思想方法`（现状不变）
- **逻辑类型标签**：`dimension=逻辑思维类型`，`subdimension=族名`，`tag=类型名`（**新增**）
- **难度**：建议给 `questions` 加一列 `difficulty INTEGER`（0–9）。这是唯一需要的 schema 改动，
  且加列非破坏（旧行 `NULL`）。**本步未改 schema**——留待生成接线时一并做并验证。

一道题最终形如：
```
statement: "一根绳子对折两次后长 30 cm，原来多长？"
knowledge_tags: [量与计量/长度]
logic_tags:     [逻辑思维类型/策略与转化/逆向倒推]
difficulty:     4
```

## 4. 落地次序（把分类接到能力上）

1. **【已落地】数据模型 → 动态标签库**：`DIM_LOGIC` + `LOGIC_CATALOGUE` + `DIFFICULTY_LEVELS`（v0.1.0a）
   现作 `tags.db` 的种子（v0.2.0a，见上方方向更新）。
2. **【已落地 v0.2.1a】出题接入动态标签库（自演化）**：单题生成（`/practice/next?generate=1`）已让 AI
   读 `tags.db` 词表 → 给题打**逻辑类型 + 知识点**标签 + **0–9 难度**（`questions.difficulty` 新列）；
   现有标签优先复用并 `bump_usage`，没有合适的就**新增标签**（`source='ai'`）并附到该题。真链路验收：
   「鸡兔同笼」→ `假设调整`+`初步代数思想`、难度 4。**待续**：批量 `/exam/generate` 仍走旧硬编码目录；
   前端 `demo_exam` 暴露 `logic_type=`/`difficulty=` 筛选。
3. **【接共识器】逻辑诊断**（目标 5/6 · 核心差异点）：多路共识推理器（`reasoner.py`）的
   **"多路分歧点"天然是诊断种子**——学生在某逻辑类型上反复出错 + 多路在该步分歧，即定位其薄弱思维。
   产出"逻辑缺陷画像"（按 13 类打分），写入 `memory.py`。
4. **【自适应推荐】**（目标 2/9）：按"逻辑缺陷画像"选**薄弱类型 + 合适难度**出下一题，替换现在
   只看 `success_rate` 的难度调节。
5. **【自我进化】**（目标 5–8）对应蓝图阶段 5（SPA，默认 dry-run）/ 阶段 6（Lattice）；高风险，
   按 `DEVELOPMENT_PLAN.md` 须在策略治理（阶段 4）就绪后再推进。

## 5. 导入自检（无网关、纯标准库）

```bash
cd backend
py -3.12 -c "from app import exam; print(len(exam.all_logic_tags()), '逻辑类型'); \
print(exam.logic_tag_info('逆向倒推')); print(exam.difficulty_label(4))"
```
应打印 13 个逻辑类型、`逆向倒推` 的族/动作/诊断信号、难度 4 的锚点。代码是唯一事实来源——
分类一旦定稿，以此自检 + `/exam` 真链路"跑起来看"为准。
